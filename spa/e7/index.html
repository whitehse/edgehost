Allowlist durability

REST allowlist edits update runtime state and are not written back to YAML. Set plugins.e7_callhome.allowlist_path for file durability across restarts (PR-10 interim). Without that path, only YAML shelves + in-memory table apply after restart.

Connection progress

    

      Live Call Home path: TCP accept → identity (Calix XML MAC or
      Junos DEVICE-CONN-INFO) → allowlist → SSH client → NETCONF →
      subscription.
      Polls GET /api/v1/e7/events?since=… every second while logged in.
      Shelves that never leave Inactive almost always fail in this log
      (wrong MAC, disabled, identity timeout, SSH auth, etc.).

Add / edit shelf

    

      PUT /api/v1/e7/shelves/{id} — path id is Calix MAC or
      Junos DEVICE-ID. For Junos Call Home, set vendor to
      junos and optionally a shared secret matching
      Junos system services outbound-ssh client … secret (HMAC of HOST-KEY).
      Secret is optional if the router does not send HOST-KEY/HMAC.
