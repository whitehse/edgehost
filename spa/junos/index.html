Juniper NETCONF Call Home

Configure devices that dial in with Junos system services outbound-ssh (DEVICE-CONN-INFO). After the initiation sequence, edgehost is the SSH client and NETCONF client on the same TCP socket (RFC 8071). Optional shared secret verifies HOST-KEY / HMAC: when the router sends them.

This UI shares the Call Home listener and allowlist with Calix E7 (./var/e7_allowlist.txt). Calix dials are handled by identity demux, not this form — manage Calix MACs on /e7/.

Lab login

POST /auth/lab-login → session cookie. Employee role required for allowlist PUT.

Add / edit Junos system

    

      PUT /api/v1/e7/shelves/{device-id} with
      vendor: "junos".
      DEVICE-ID must match
      set system services outbound-ssh client … device-id on the router.
      Shared secret is optional and must match the router
      secret statement when HOST-KEY/HMAC are sent.
    


    

      
      
      
      
      
    


    

      
      
      
    

    


    

      Router configuration sketch
      
set system services netconf ssh
set system services outbound-ssh client edgehost device-id pe1.lab
set system services outbound-ssh client edgehost services netconf
set system services outbound-ssh client edgehost servers <nms-ip> port 4334
# optional HMAC of HOST-KEY:
# set system services outbound-ssh client edgehost secret "shared-secret"
